# Hachi UI Examples

This directory intentionally keeps only a few lightweight examples.

- `wireframe-briefwell-onboarding.svg`: monochrome left-to-right onboarding flow with external screen contracts and explicit transition triggers.
- `briefwell-boardroom.svg`: high-fidelity boardroom-style business application screen using a design-system pass.
- `nagare-ui-element-plan.md`: semantic UI planning example that maps an AI workflow product brief into decisions, information units, screen patterns, UI elements, area budget, and evidence needs.
- `nagare-screen-family-plan.md`: screen-family continuity example showing how a selected direction carries across Work Home, Work Detail, Domain Knowledge Settings, and Artifact Editor Dialog.
- `nagare-conversation-replay-test.md`: planning regression example for direction varianting, user correction handling, auditability, and wide prompt/rubric editing.

Generated screenshots, exploratory variants, per-project `requirements.md`, `UI_PLAN.md`, `DESIGN.md`, caches, and local test output should stay untracked unless they are promoted to curated examples.
